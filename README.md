## Project Overview

This system takes a reading comprehension passage and automatically:
- Predicts which of four answer options is correct (Model A)
- Generates 3 plausible wrong-answer distractors (Model B)
- Extracts 3 graduated hints from the article (Model B)
- Presents everything in a 4-screen Streamlit web interface

**Dataset:** RACE (ReAding Comprehension from Examinations) — 87,866 training rows  
**Constraint:** Classical ML only — no neural networks, no transformers, no BERT/LSTM

## How to Run (in this exact order)

```bash
# Activate venv first every time you open a new terminal
venv\Scripts\activate

# Step 1 — Preprocess the RACE dataset (~5 min)
# Loads a SINGLE train.csv and performs 80-10-10 stratified split (seed=42)

# Step 2 — Train Model A: answer verification (~20–40 min on CPU)

# Saves all models to models/model_a/traditional/

# Step 2b — Train Question Generator (~10–15 min)
# Builds TF-IDF sentence vectorizer + LR ranker for template-based QG
# Saves qg_tfidf.pkl, qg_ranker.pkl to models/model_a/traditional/
python src/model_a_generate.py

# Step 3 — Train Model B: distractors + hints (~10–15 min)
# Trains Word2Vec, builds frequency table, saves to models/model_b/traditional/
python src/model_b_train.py

# Step 4 — Evaluate all models on test set + run GridSearchCV (~10 min)
# Saves evaluation_results.csv, qg_metrics.csv (BLEU/ROUGE/METEOR),
# benchmark_comparison.csv (vs BERT/T5), and confusion matrix PNGs
python src/evaluate.py

# Step 5 — Launch the Streamlit web app
streamlit run ui/app.py
```

The app will open automatically at `http://localhost:8501`

## Model A — Answer Verification

**Task:** Given an article + question + one answer option, predict whether that option is correct (binary: is_correct = 0 or 1). The option with the highest confidence score across all 4 options is selected as the predicted answer.

**Features used:**
- One-Hot Encoding (CountVectorizer, binary=True, max_features=10,000)
- Cosine similarity between article and answer option
- Numeric features: article word count, question word count

**Models trained:**

| Model | Type | Notes |
|-------|------|-------|
| Logistic Regression | Supervised | Full dataset, class_weight=balanced |
| LinearSVC | Supervised | Full dataset |
| ComplementNB | Supervised | Full dataset, best for imbalanced text |
| Random Forest | Supervised | 30k-row subset |
| XGBoost | Supervised | 30k-row subset |
| KMeans | Unsupervised | 10k-row subset, n_clusters=4 |
| GMM | Unsupervised | 10k-row subset + TruncatedSVD(50) |
| LabelPropagation | Semi-supervised | 5k rows, 10% labeled |
| Ensemble (LR+NB) | Ensemble | Soft voting, no retraining |

---

## Model B — Distractor & Hint Generation

**Distractor Generation** (3 wrong-but-plausible alternatives):
1. **OHE + Cosine Similarity** (primary): finds article words with medium similarity to the correct answer — not too close (identical) and not too far (random)
2. **Word2Vec** (semantic): finds semantically similar words outside the article
3. **Frequency Matching** (statistical): finds corpus words with similar frequency as the answer

**Hint Extraction** (graduated, 3 levels):
- Scores each article sentence by keyword overlap with the question
- Hint 1 (vague) = 3rd-best sentence
- Hint 2 (moderate) = 2nd-best
- Hint 3 (revealing) = best-matching sentence

## Key Design Decisions

- **Sparse matrices throughout:** All feature matrices stay as `scipy.sparse.csr_matrix`. Never call `.toarray()` — it would cause MemoryError on 350k rows.
- **No data leakage:** `fit_transform()` is called only on training data. Val and test sets use `transform()` only.
- **OHE = primary feature:** `CountVectorizer(binary=True)` is the main feature method as required by the rubric.
- **Question-level accuracy:** The meaningful metric for this task. For each group of 4 rows (one question), the model must rank the correct option highest.

## Troubleshooting

| Error | Fix |
|-------|-----|
| `Missing package: ...` | Run `venv\Scripts\activate` then `pip install -r requirements.txt` |
| `Could not find train.csv` | Place `train.csv`, `dev.csv`, `test.csv` in `data/raw/` |
| `Missing files in data/processed/` | Run `python src/preprocessing.py` first |
| `Missing model files` | Run `python src/model_a_train.py` first |
| `ValueError: X has N features but model expects M` | The OHE vectorizer must match — load from `models/model_a/traditional/ohe_vectorizer.pkl` |
| Streamlit `st.image()` error | Upgrade: `pip install streamlit --upgrade` |
